Accelerator inner-workings

Each accelerator follows a simple structure, contained in a seperate directory, which helps you merge the parts you like from multiple accelerators, and helps keep things consistent for your users who are evaluating prototypes.

The folder structure of an accelerator follows this form:
  • root folder: This contains all the files that are unique to this accelerator. Alpha-in-a-box is built using Jekyll, which uses templates to minimise duplication, and common components (such as the HTML HEAD element declarations, and the accelerator navigation footer, are implemented using Jekyll templates)
    • index.html: Outlines the accelerator's business and technical approach, and highlights the key areas that implementers should be aware of when making adapting the accelerator
    • reset.html: Used to initialise and reset the accelerator between runs, allowing users to go back and try things again
    • start.html: The entry page of the accelerator, which is where your users will start their journey through the prototype
    • style.css: Custom styling that you'd like to apply to the prototype
